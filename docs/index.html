The Solution

A Structured Escalation Workflow That Doesn't Give Up

Built inside GoHighLevel, this workflow triggers when a lead goes unresponsive. Instead of stopping, it escalates through up to 5 backup contacts — checking fields, firing webhooks, and moving on until someone responds.

  • ✓ Follows up beyond the primary contact using pre-mapped relative fields
  • ✓ Checks whether each contact has a valid phone number or email before sending
  • ✓ Triggers a webhook at each step to execute external actions or notify systems
  • ✓ Automatically stops when no further backup contacts are available
  • ✓ Links each relative contact back to the parent lead record for full visibility

How It Works

Step-by-Step Workflow Logic

The workflow follows a sequential escalation pattern — each step only executes if the previous contact was unreachable or unavailable.

1

Trigger: Contact Created or Stage Changed

📋 workflow-overview.png — trigger setup

Triggers when a new lead is added or moved into a "No Response" pipeline stage.

Trigger: Pipeline Stage → "No Response"
2

Conditional Check: Primary Contact Fields

Checks if the contact has a valid phone or email. If both are missing, skips straight to the first backup contact.

IF phone IS EMPTY → skip to Relative 1
3

Webhook Execution: Primary Contact

🔗 webhook-config.png — endpoint & payload mapping

A webhook fires with the contact's name, phone, email, and lead context — triggering outreach in the connected system.

POST → webhook.site / n8n / Zapier endpoint
4

Escalation to Relative 1 → Relative 5

📈 escalation.png — Relative 1 through Relative 5 chain

No response? The system checks Relative 1's fields, fires a webhook if valid, then moves to Relative 2 — repeating through Relative 5.

Relative 1 → check → fire → wait → Relative 2 → ...
5

Stop Condition: No More Contacts

All contacts exhausted? The workflow ends cleanly. The lead gets flagged for manual review.

IF Relative 5 IS EMPTY → End Workflow

Technical Implementation

What's Under the Hood

For those interested in the technical implementation — native GHL workflow features combined with external webhooks to create a fully automated escalation process.

🏛

Conditional Branching

Each escalation step uses GHL's IF/ELSE branch logic to evaluate whether contact fields are populated before proceeding.

🔄

Sequential Workflow Logic

Steps execute in order with built-in wait periods between attempts, ensuring structured, non-spammy outreach at each escalation level.

🔗

Webhook / API Requests

Each escalation fires a POST request to a configured endpoint, passing contact data in real time to trigger downstream actions.

👥

Dynamic Field Mapping

Name, phone, email, relationship type, and parent contact ID are all mapped dynamically from GHL custom fields into the payload.

📄

JSON Payload Structure

A standardized payload is sent at every step, keeping the receiving system consistent regardless of which relative is being contacted.

🔗

Parent Contact Linking

Each relative record is linked to the original lead using a custom parent contact ID field, maintaining full relationship visibility.

⚠️ Edge case handled: If a webhook fails mid-escalation, the workflow can be extended with retry logic or error-tagging steps that flag the lead for manual review — ensuring no silent failures in production.
